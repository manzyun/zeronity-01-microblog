# Agent Instructions & Constraints

このプロジェクトは、AIエージェントと人間（まんじゅん）が協働して開発するマイクロブログサービスです。

## 核心的原則：コミュニティへの敬意と持続可能性

* **外部リポジトリの保護**: プロジェクトで利用しているフレームワーク（Flask, Svelte等）やライブラリ自体のソースコードを改変しないこと。
* **上流への配慮**: ライブラリ側に修正が必要だと判断した場合は、必ずユーザーに報告し、議論すること。AIが勝手にパッチを作成し、外部へ申請するような振る舞いは厳禁とする。パッチ提案の責任と判断は人間が負う。

## 設計・実装方針

* **ドメイン駆動設計 (DDD) の尊重**: 
    - 技術詳細（DBやWebフレームワーク）よりも「ドメイン（投稿、フォロー、ActivityPubの配送等）」の整合性を優先する。
    - クリーンアーキテクチャまで厳格にせずとも、ビジネスロジックがフレームワークに密結合しないよう配慮すること。
* **プロトコル**: 将来的な **ActivityPub** への対応を視野に入れ、ActorやActivity（Create, Follow等）の概念をデータ構造に含めること。
* **可視化**: 設計の意図を伝える際は、`docs/` 配下に Mermaid 形式で図解（クラス図、シーケンス図等）を出力すること。拡張子は `.mmd` とする。
    - 文書として記す場合は `.md` とする

## 開発サイクルとテスト

* **テスト駆動の推奨**: 
    - 「テストの実行方法」は `README.md` または `Makefile` に明記する。
    - AIがコードを生成する際は、必ず対応するテストコード（標準モジュール『unittest』, 『unittest.mock』 等）をセットで提案すること。
    - リリース可能な状態を維持するため、既存のテストを壊す変更は行わないこと。

## 環境・ツール

* **OS**: Ubuntu
* **Editor**: DOOM Emacs (User is a power user)
* **Stack**: Python (Flask) / JavaScript (Svelte) / SQLiteDocument Title
